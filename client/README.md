# Краткий отчёт по реализации веб-интерфейса парсера документов

## 1. Использованные технологии

Все технологии имеют лицензию **MIT**, что соответствует требованиям технического задания.

| Технология | Версия | Лицензия | Назначение |
|---|---|---|---|
| **React** | 18.x | MIT | UI-фреймворк, компонентная архитектура SPA |
| **React DOM** | 18.x | MIT | Рендеринг React-компонентов в браузере |
| **Axios** | 1.15.0 | MIT | HTTP-клиент для асинхронных запросов к backend |
| **Bootstrap** | 5.x | MIT | CSS-фреймворк для стилизации интерфейса |

***

## 2. Архитектура решения

**Компоненты:**
- `App.jsx` — корневой компонент, передаёт состояние дочерним компонентам
- `Uploader.jsx` — выбор, валидация и отображение списка файлов
- `CheckBox.jsx` — настройки парсера и отправка задачи
- `ReadyToGrabFiles.jsx` — отображение статуса задачи и скачивание результата
- `useParserSettings` — хук с логикой: отправка задачи, polling статуса, скачивание, localStorage

***


## 3. Обработка ошибок

Все ошибки отображаются в графическом интерфейсе в виде человекочитаемых сообщений.
**Клиентская валидация (до отправки на сервер):**
- Не выбраны файлы → сообщение в форме
- Не выбран ни один формат документа → сообщение в форме
- Файл пустой или превышает 50 МБ → сообщение в форме
- Недопустимое расширение файла → сообщение в форме

**Серверные ошибки (HTTP):**

| Код | Сообщение пользователю |
|---|---|
| 400 | Ошибка запроса — проверьте данные формы |
| 413 | Файл слишком большой (максимум 50 МБ) |
| 415 | Недопустимый формат файла |
| 422 | Некорректные параметры запроса |
| 404 | Задача не найдена |
| 409 | Задача ещё не завершена |
| 500 | Внутренняя ошибка сервера |
| 503 | Сервер временно недоступен |

***

## 4. Сохранение состояния

Для обеспечения устойчивости к случайным перезагрузкам страницы используется `localStorage`:

- **Настройки парсера** — автоматически сохраняются при каждом изменении и восстанавливаются при открытии
- **Последняя задача** — `task_id` сохраняется после создания, что позволяет скачать результат после перезагрузки страницы
